# /dev:patch

超轻量补丁模式。零文档、零 git、零团队。架构师快速分析，开发者直接实现。

## 用法

```bash
/dev:patch <改动描述>
```

## 参数

| 参数 | 说明 |
|------|------|
| `改动描述` | 要做的改动描述，支持中文 |

无 `--git`/`--no-git` 参数。patch 模式不使用 git。

## 适用场景

- 改个配置、修个 typo、调个样式
- 小 bug fix，原因明确
- 已知怎么改，只需要执行
- 不值得走完整工作流的任何改动

## 不适用

- 涉及 3+ 文件的改动 -- 用 `/dev:fix`
- 需要新分支、PR -- 用 `/dev:feat` 或 `/dev:fix`
- 需要测试设计 -- 用 `/dev:feat`

## 执行流程

```
/dev:patch 修复登录按钮在移动端不显示的问题
    |
Step 1: 架构师快速分析 (opus) -> 方案摘要（口头，不写文件）
    |
Step 2: 向用户展示方案，确认或调整
    |
Step 3: 开发者实现 (sonnet) -> 直接改代码
    |
Step 4: 运行验证（build/lint/test，如果有配置）
    |
Done. 不创建分支，不生成文档，不写 .dev/ 目录。
```

预期耗时：2-3 分钟。

## 与其他命令的区别

| | `/dev:patch` | `/dev:fix` | `/dev:feat` |
|---|---|---|---|
| Git 分支 | 否 | 可选（默认是） | 是 |
| 文档生成 | 无 | Bug Report + ACCEPTANCE | 全套 |
| Agent 数 | 2 | 3 | 5-7 |
| 人类确认 | 1 次 | 1 次 | 1 次（产品讨论） |
| 适用改动量 | 1-2 文件 | 1-5 文件 | 3-8 文件 |
| 预期耗时 | 2-3 min | 5-10 min | 15-45 min |

## 注意事项

- 不创建分支、不生成文档、不写 `.dev/` 目录
- 如果改动涉及 3+ 文件，建议使用 `/dev:fix` 或 `/dev:feat`
