# /dev:start

启动多 Agent 迭代开发工作流，覆盖从需求讨论到验收的完整流程。

## 用法

```bash
/dev:start <功能描述>
```

## 参数

- `功能描述`: 要开发的功能描述，支持中文。如 "添加用户登录功能"

## 执行流程

### 初始化阶段（一次性）

| Step | 内容 | 产出 |
|------|------|------|
| 1 | 解析输入 | - |
| 2 | 自动创建 git 分支（feat/fix/refactor） | 新分支 |
| 3 | 初始化 `.dev/plan/<branch>/` 状态目录 | TASK-LOG.md |
| 4 | **spawn dev-product** — 产品讨论（唯一人类介入点） | PRD.md + 线框图 |
| 5 | **spawn dev-planner** — 规划（基于 PRD） | PLAN.md |
| 6 | **spawn dev-tester** — 测试设计（先设计再开发） | TEST-DESIGN.md |

### 开发循环（逐任务）

| Step | 内容 | 产出 |
|------|------|------|
| 7 | **spawn dev-developer** — 开发（每次 1 个任务） | 代码 + 提交 |
| 8 | 执行验证命令 — 验证当前任务 | 测试结果 |
| 9 | 决策 — 全部完成则进入验收，否则回到 Step 7 | 进度更新 |

### 收尾阶段

| Step | 内容 | 产出 |
|------|------|------|
| 10 | 生成验收说明书 | ACCEPTANCE.md |

## 分支类型判断

根据功能描述关键词自动判断：

- 包含 "修复"、"bug"、"fix"、"错误" → `fix/<slug>`
- 包含 "重构"、"refactor"、"优化"、"清理" → `refactor/<slug>`
- 默认 → `feat/<slug>`

## 经验记录

每个阶段完成后自动在后台 spawn recorder 记录经验，不阻断主流程：

| 阶段 | 触发时机 |
|------|---------|
| product | Step 4 产品讨论完成后 |
| planning | Step 5 规划完成后 |
| test_design | Step 6 测试设计完成后 |
| development | Step 7 开发完成后 |
| verification | Step 8 验证通过后 |
| acceptance | Step 10 验收完成后 |

## 输出示例

```
================================================================
 DEV WORKFLOW STARTED
================================================================
 功能: 添加用户登录功能
 分支: feat/user-login
================================================================
```

完成时：

```
================================================================
 DEV WORKFLOW COMPLETE
================================================================
 功能: 添加用户登录功能
 分支: feat/user-login
 任务: 完成 5/5
 测试: 通过 12/12
 提交: 8 个
 验收说明书: .dev/plan/feat/user-login/ACCEPTANCE.md

 下一步:
   验收测试  — 按验收说明书确认功能
   /dev:status   — 查看工作流状态
   /git:finish   — 合并到 develop
================================================================
```

## 注意事项

- 工作流是 **先设计再开发**，不是边写边想
- 工作流是 **逐任务循环**，一个任务完成验证后才开发下一个
- 主命令 **只做调度**，所有技术工作委托给子 Agent
- 不要自动执行 `/git:finish`，由用户自行决定合并时机
- 如果中途中断，使用 `/dev:resume` 恢复

## 相关命令

- [/dev:status](./status) — 查看工作流状态
- [/dev:resume](./resume) — 恢复中断的工作流
- [/git:finish](../git/finish) — 完成并合并分支
