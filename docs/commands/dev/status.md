# /dev:status

查看当前开发工作流状态和进度。

## 用法

```bash
/dev:status
```

## 功能

- 显示当前分支的工作流状态
- 展示任务进度条和任务列表
- 显示测试通过情况
- 给出下一步操作建议

## 输出格式

```
================================================================
 DEV WORKFLOW STATUS
================================================================
 分支: feat/user-login
 状态: 进行中

 任务进度: 2/5 完成
 ||||||||||||||||||||||||.................... 40%

 测试: 通过 6/6

 提交数: 4
================================================================

 任务详情:
 | T-01 | 用户登录表单界面   | 完成 | a1b2c3d |
 | T-02 | 登录 API 对接      | 完成 | e4f5g6h |
 | T-03 | 表单验证逻辑       | 待执行 | -      |
 | T-04 | 错误状态处理       | 待执行 | -      |
 | T-05 | 记住密码功能       | 待执行 | -      |

 下一步:
   /dev:resume  -- 继续执行未完成的任务
   /git:status  -- 查看 git 分支状态
   /git:finish  -- 合并到 develop
```

## 信息来源

命令读取 `.dev/plan/<branch-name>/` 目录下的状态文件：

- **PLAN.md** -- 任务列表和总数
- **TASK-LOG.md** -- 执行记录表格
- **TEST-REPORT.md** -- 测试结果（如存在）

## 相关命令

- [/dev:start](./start) -- 启动工作流
- [/dev:resume](./resume) -- 恢复工作流
- [/git:status](../git/status) -- 查看 git 分支状态
