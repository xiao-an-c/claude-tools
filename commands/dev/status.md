# /dev:status — 查看工作流状态

## 执行流程

### 1. 获取当前分支

```bash
CURRENT_BRANCH=$(git branch --show-current)
```

如果不是 `feat/*`、`fix/*`、`refactor/*` 分支，提示 "当前分支没有活跃的开发工作流"。

### 2. 检查状态目录

检查 `.dev/plan/<current-branch>/` 是否存在。

如果不存在，提示 "未找到工作流状态文件。使用 /dev:start 启动工作流。"

### 3. 读取并展示状态

读取以下文件并展示：

**PLAN.md** — 展示任务总数和列表

**TASK-LOG.md** — 展示执行记录表格

**TEST-REPORT.md** — 如果存在，展示测试结果

### 4. 展示格式

```
================================================================
 DEV WORKFLOW STATUS
================================================================
 分支: <branch>
 状态: <进行中/已完成>

 任务进度: X/Y 完成
 ┃████████░░░░░░░░░░░░░░░░░░░░░░░ 40%

 测试: 通过 X/Y (如已有测试)

 提交数: N
================================================================

 任务详情:
 | T-01 | 功能标题     | 完成 | abc1234 |
 | T-02 | 功能标题     | 完成 | def5678 |
 | T-03 | 功能标题     | 待执行 | -      |

 下一步:
   /dev:resume  — 继续执行未完成的任务
   /git:status  — 查看 git 分支状态
   /git:finish  — 合并到 develop
```
